# 动漫生产管理系统 - 规格文档 v3

## 1. 项目概述

- **项目名称**: 动漫工作室 (Anime Studio)
- **核心功能**: 动漫生产全流程管理工具，包含配置管理、项目创建、分镜制作、生图生视频
- **目标用户**: 动漫工作室团队成员

## 2. 技术栈

- **框架**: Tauri 2.x (Rust后端 + Web前端)
- **前端**: React 18 + TypeScript + Tailwind CSS
- **构建**: Vite
- **状态管理**: React Context + useReducer
- **目标平台**: Windows 10

## 4. 功能规格

### 4.1 设置面板 (左侧菜单进入)

**Tab 1: 基础**
| 字段 | 类型 | 说明 |
|------|------|------|
| 正向提示词 | 多行文本 | 默认提示词模板 |
| 负向提示词 | 多行文本 | 不希望出现的元素 |
| 风格 | 文本输入 | 可填写任意风格 |
| 尺寸 | 文本输入 | 可填写任意尺寸如 1920x1080 |

**Tab 2: 模型预设**
| 字段 | 类型 | 说明 |
|------|------|------|
| 预设名称 | 单行输入 | 如"默认GPT"/"Gemini生图"等 |
| API地址 | 单行输入 | 留空则使用默认 https://api.jxincm.cn |
| API KEY | 密码输入 | 必填，API密钥 |
| 模型名称 | 单行输入 | 如 gpt-4o / gemini-2.0-flash-preview-image-generation |
| 预设列表 | 可增删 | 保存多个预设 |

**Tab 3: 咒语**
| 字段 | 类型 | 说明 |
|------|------|------|
| 分镜咒语 | 多行文本 | AI分镜提示词模板 |
| 改文咒语 | 多行文本 | 文本改写提示词 |
| 图片咒语 | 多行文本 | 图片生成提示词模板 |
| 动漫咒语 | 多行文本 | 视频/动漫生成提示词模板 |

### 4.2 项目列表 (侧边菜单)

- 显示所有已创建项目
- 每个项目显示：名称、创建时间、分镜数
- 点击进入项目详情

### 4.3 新建项目

- 项目描述

### 4.5 API调用逻辑

**API Base URL**: `https://api.jxincm.cn`

#### 4.5.1 文本处理 (改文/分镜/提示词)
- **端点**: `POST /v1/chat/completions`
- **请求**:
```json
{
  "model": "gpt-4o",
  "messages": [{"role": "user", "content": "提示词+内容"}],
  "temperature": 0.7
}
```
- **响应**: `choices[0].message.content`

#### 4.5.2 图片生成
- **端点**: `POST /v1beta/models/gemini-2.0-flash-preview-image-generation:generateContent`
- **请求**:
```json
{
  "contents": [{"parts": [{"text": "prompt"}]}],
  "generationConfig": {"responseModalities": ["TEXT", "IMAGE"]}
}
```
- **响应**: 包含base64图片数据

#### 4.5.3 视频生成
- **端点**: `POST /v1/video/create`
- **请求**:
```json
{
  "model": "veo3.1-fast",
  "prompt": "视频提示词",
  "enhance_prompt": true
}
```
- **响应**: `{"id": "veo3.1-fast:xxx", "status": "pending"}`

#### 4.5.4 查询视频状态
- **端点**: `GET /v1/video/query?id=veo3.1-fast:xxx`
- **响应**: `{"status": "completed", "video_url": "https://..."}`

### 4.6 批量生成

- **并发数设置**: 1-100，可手动输入
- **批量生图**: 根据并发数并行调用图片生成API
- **批量视频**: 根据并发数并行调用视频生成API（需要轮询状态）

### 4.7 上下文提示词构建

生成图片/视频提示词时，提交内容包含：
1. 全剧概要
2. 当前分镜前后各10行上下文
3. 当前分镜内容

格式：
```
【全剧概要】
{全剧概要内容}

【前后文】
{前10行内容}
---
{当前分镜内容}
---
{后10行内容}

【当前分镜】
{当前分镜内容}
```
